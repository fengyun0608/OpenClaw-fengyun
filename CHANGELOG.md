# 更新日志 (CHANGELOG)

## [1.3.0] - 2026-03-12

### 新增
- **文件发送支持**：完整支持发送各类文件到 QQ
  - 图片：jpg, png, gif, bmp, webp, ico, tiff, svg
  - 文档：pdf, doc, docx, pptx, txt, md
  - 音频：mp3, wav, m4a, ogg, flac, amr
  - 视频：mp4, mkv, mov, avi, wmv, flv
  - 压缩：zip, rar, 7z, tar, gz, bz2
  - 其他：exe, apk, iso, sqlite, json, xml, html, php, sh 等
- **智能文件类型识别**：通过文件头魔数自动识别文件类型
- **文件名格式**：`扩展名.扩展名`（如 `pdf.pdf`, `mp3.mp3`）

### 修复
- **适配器协议兼容**：完全适配 openclaw-xrk-bridger 适配器协议
  - 支持 `mediaUrls` 字段（图片）
  - 支持 `files` 字段（其他文件）
  - 支持 `selfId` 多账号路由
- **base64 URL 处理**：修复 `base64://` 格式解析问题
- **群聊限制**：群聊模式只发送文本，跳过媒体/文件发送（适配器不支持）

### 优化
- **消息分开发送**：文本和媒体分开发送，避免 NTQQ 富媒体传输失败
- **独立错误处理**：单个文件发送失败不影响其他内容
- **详细日志**：每个发送操作都有独立日志

## [1.2.0] - 2025-03-10

### 新增
- **帮助指令**：`#op帮助` 或 `op帮助` 查看帮助信息
- **指令优化**：`#` 符号可选，支持 `#op添加` 和 `op添加` 两种格式
- **大小写不敏感**：支持 `#OP添加`、`#Op添加`、`#op添加` 等各种写法

### 修复
- 修复正则表达式匹配问题
- 修复指令大小写敏感问题

## [1.1.0] - 2025-03-10

### 新增
- **授权系统**：添加用户授权管理功能
  - `#op添加@用户` - 添加授权用户（仅主人可用）
  - `#op删除@用户` - 删除授权用户（仅主人可用）
  - `#op列表` - 查看授权列表（仅主人可用）
- **权限控制**：只有主人和授权用户才能使用机器人
- **配置持久化**：授权用户列表保存在 `data/xrk-bridge-op.json`
- **冷却机制**：3秒冷却时间，防止消息刷屏

### 优化
- 简化代码结构，删除冗余的危险检测逻辑
- 优化消息转发日志输出
- 消息标识区分：主人消息显示 `[主人✓]`，授权用户显示 `[授权]`

### 修复
- 修复导入路径问题
- 修复插件加载问题

## [1.0.0] - 2025-03-10

### 新增
- 初始版本发布
- 实现 XRK-Yunzai 与 OpenClaw 的 WebSocket 桥接
- 支持群艾特消息自动转发
- 支持私聊消息自动转发
- 支持流式消息发送
- 支持多媒体消息（图片、视频、音频、文件）
- HTTP API 接口（状态查询、消息发送）

### 技术实现
- `bridge.js`: WebSocket 服务端，处理连接和消息分发
- `forward.js`: 消息转发插件，监听 QQ 消息并转发到 OpenClaw
- `index.js`: 插件入口，自动加载 apps 目录下的模块

### 依赖
- ws: WebSocket 通信
- XRK-Yunzai 框架
- OpenClaw (小龙虾) 及 xrk-agt-bridge 扩展
